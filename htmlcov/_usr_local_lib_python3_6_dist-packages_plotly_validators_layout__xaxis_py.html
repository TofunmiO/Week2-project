244 scaleanchor 

245 If set to another axis id (e.g. `x2`, `y`), the 

246 range of this axis changes together with the 

247 range of the corresponding axis such that the 

248 scale of pixels per unit is in a constant 

249 ratio. Both axes are still zoomable, but when 

250 you zoom one, the other will zoom the same 

251 amount, keeping a fixed midpoint. `constrain` 

252 and `constraintoward` determine how we enforce 

253 the constraint. You can chain these, ie `yaxis: 

254 {scaleanchor: *x*}, xaxis2: {scaleanchor: *y*}` 

255 but you can only link axes of the same `type`. 

256 The linked axis can have the opposite letter 

257 (to constrain the aspect ratio) or the same 

258 letter (to match scales across subplots). Loops 

259 (`yaxis: {scaleanchor: *x*}, xaxis: 

260 {scaleanchor: *y*}` or longer) are redundant 

261 and the last constraint encountered will be 

262 ignored to avoid possible inconsistent 

263 constraints via `scaleratio`. Note that setting 

264 axes simultaneously in both a `scaleanchor` and 

265 a `matches` constraint is currently forbidden.
